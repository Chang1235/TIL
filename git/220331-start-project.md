#git

##commit Convention
- 커밋 제목은 50자 이내로 요약하여 작성한다.
- 제목과 내용사이 한 칸 띄운다.
- prefix를 사용하여 한 눈에 커밋의 용도를 알기 쉽게한다.
  - feat: 기능 개발 관련
  - fix: 오류 개선 혹은 버그 패치
  - docs: 문서화 작업
  - test: test 관련
  - fix: bug-fix
  - refactor : refactoring
  - conf: 혼경설정 관련
  - build: 빌드 관련
  - ci :Continus Integration
